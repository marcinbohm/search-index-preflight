# v0.0.2-prealpha release notes

SearchIndexPreflight `v0.0.2-prealpha` is an experimental, source-only
pre-alpha release focused on repository organization, installation
documentation, contributor readiness, and public OSS hygiene after
`v0.0.1-prealpha`.

It is intended for evaluation and feedback. It is not production-ready as a
required CI gate, and it does not include binary release artifacts.

## What changed

### Installation and evaluation

- The tagged `go install` workflow was verified against `v0.0.1-prealpha`.
- The README and getting-started guide document the supported module path and
  version check for that source-only pre-release.
- The documentation keeps source evaluation, synthetic fixtures, and local CLI
  review as the supported evaluation path.

### Repository and roadmap organization

- Repository description, discovery topics, and release/CI badges were checked
  for the public project presentation.
- GitHub labels and milestones now organize release work, areas, priorities,
  status, and community participation.
- The public roadmap issue and epic issues provide an issue-level index for
  planned work; they are planning aids, not commitments that every future item
  will ship in this release.
- `docs/PROJECT_BACKLOG.md` is the broader planning source of truth, while
  GitHub Issues remain the execution layer for actionable work.
- A public feedback entry point is available for real Elasticsearch/OpenSearch
  schema failure cases.

### Contributor and community readiness

- The repository includes Contributor Covenant 3.0 and a private project contact
  for conduct or security-sensitive reports.
- Contributor guidance now matches the current issue-label taxonomy.
- Existing issue and pull-request templates remain available, with issue-template
  labels aligned to labels that exist in the repository.
- Contributor and issue guidance continues to require synthetic examples and
  warns against publishing credentials, production mappings, customer data,
  private logs, or internal service names.

## Current CLI scope

There are no new commands or rules in this release. The current public commands
remain:

- `lint`
- `diff`
- `rules list`
- `explain <RULE_ID>`
- `version`

The current rules remain:

- `SIL001`
- `SIL002`
- `SIL003`
- `DIF001`
- `DIF002`
- `DIF003`

## Not included

- Release binaries or other published packages.
- Production-ready CI-gate guarantees.
- New lint or diff rules, including `DIF004`.
- Field option diff behavior.
- Config loading.
- Suppressions.
- Baseline mode.
- YAML input.
- Markdown or SARIF output.
- A GitHub Action.
- Cluster doctor functionality.
- Migration/versioning implementation.

These remain future work and are not part of the current CLI contract.

## Feedback wanted

Please use the [public feedback issue](https://github.com/marcinbohm/search-index-preflight/issues/13)
to share focused, synthetic or minimized examples. No feedback results are
claimed here; the issue remains an open entry point for discovery.

In particular, feedback on these questions is useful:

- Which real Elasticsearch/OpenSearch schema rollout failures should be checked
  next?
- Which diff or preflight checks would provide the most value during review?
- Are the installation and repository guidance clear?
- Is the current CLI output useful during pull-request review?
